15 * A byte-addressable ring buffer FIFO implementation.
16 *
17 * The ring buffer's head pointer points to the starting location
18 * where data should be written when copying data *into* the buffer
19 * (e.g., with ringbuf_read). The ring buffer's tail pointer points to
20 * the starting location where data should be read when copying data
21 * *from* the buffer (e.g., with ringbuf_write).
22 */

115 /*
116 * Beginning at ring buffer dst's head pointer, fill the ring buffer
117 * with a repeating sequence of len bytes, each of value c (converted
118 * to an unsigned char). len can be as large as you like, but the
119 * function will never write more than ringbuf_buffer_size(dst) bytes
120 * in a single invocation, since that size will cause all bytes in the
121 * ring buffer to be written exactly once each.
122 *
123 * Note that if len is greater than the number of free bytes in dst,
124 * the ring buffer will overflow. When an overflow occurs, the state
125 * of the ring buffer is guaranteed to be consistent, including the
126 * head and tail pointers; old data will simply be overwritten in FIFO
127 * fashion, as needed. However, note that, if calling the function
128 * results in an overflow, the value of the ring buffer's tail pointer
129 * may be different than it was before the function was called.
130 *
131 * Returns the actual number of bytes written to dst: len, if
132 * len < ringbuf_buffer_size(dst), else ringbuf_buffer_size(dst).
133 */
134 size_t ringbuf_memset(ringbuf_t* dst, int c, size_t len);
